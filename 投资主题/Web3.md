# 投资主题：Web3 / Agentic Payment

> **定义**：关注 AI Agent 与区块链的交叉点——不是"Crypto 硬蹭 AI"，而是"AI 需要 Crypto 来完成自主经济行为"  
> **核心信念**：Agent 支付只是表象，Agent 银行才是终局；2026 是 Agent 参与经济体的关键一年  
> **更新时间**：2026-04-26

---

## 一、从第一性原理出发

### 1.1 为什么 AI 需要 Crypto？

**核心问题**：当 AI Agent 开始自主行动时，它需要什么？

| 需求 | 传统互联网方案 | 为什么不够 | Crypto 方案 |
|------|-------------|-----------|------------|
| **身份认证** | 邮箱/手机号 + 密码 | 无跨国界通用性，KYC 门槛高 | 链上地址 = 全球通用身份 |
| **支付结算** | 银行卡/PayPal | 跨境慢、手续费高、Agent 无法开户 | 稳定币 = 7x24 全球即时结算 |
| **信任机制** | 平台担保 | 平台单点故障、审查风险 | 智能合约 = 代码即信任 |
| **资产托管** | 银行账户 | 银行不给 Agent 开户 | 自托管钱包 = Agent 自主掌控资产 |
| **价值交换** | 法币 | 小额支付不经济 | 微支付协议（x402）= 按次付费 |

**第一性结论**：AI Agent 是"没有国籍、没有银行账户、24 小时在线"的经济体参与者，传统金融基础设施无法服务它们，Crypto 是天然的基础设施。

### 1.2 Agent 银行 vs Agent 支付

> "Agent 支付很热，但如果你只看支付这一层，其实是把问题想小了。" — Hazel Hu，支无不言

| 层级 | 对应传统银行 | Agent 银行的问题 | 当前成熟度 |
|------|-------------|-----------------|-----------|
| **Identity（身份）** | 存款/KYC | Agent 归属于谁？如何证明"我是谁"？ | ⭐⭐ 早期 |
| **Liquidity（流动性）** | 信贷 | Agent 没钱怎么办？ | ⭐⭐ 非常早期 |
| **Security Guardrail（安全护栏）** | 风控/反洗钱 | Agent 做的事是否正确、是否安全？ | ⭐⭐ 早期 |
| **Discoverability（发现层）** | 汇/搜索 | Agent 如何发现需要的服务？ | ⭐ 概念期 |

**投资逻辑**：支付层最成熟（已发生），但价值最大的在信贷层和安全层（还没人认真做）。

## 二、细分方向发散

### 方向 1：Agent 支付协议 / 基础设施

**核心逻辑**：
- HTTP 402 = "Payment Required"，x402 让这个动作标准化
- 不是"做一个更好的 Stripe"，而是"做一个 Agent 能用的 Stripe"

**竞争格局**：

| 协议/产品 | 背景 | 路线 | 优势 | 风险 |
|----------|------|------|------|------|
| **x402** | Coinbase 推出，已捐赠给 Linux 基金会 | 开放协议，permissionless | 去中心化、可组合、社区驱动 | 交易量还很小 |
| **MPP (Machine Payments Protocol)** | Stripe 发布 | Stripe 生态内闭环 | 商户网络大、合规强 | 中心化、硅谷反垄断心态 |
| **Tempo** | Stripe 主导的专业链 | 高性能结算 | Stripe 品牌背书 | 独立性受质疑 |
| **传统卡支付** | Visa/Mastercard | KYC + 银行卡 | 用户熟悉、合规完善 | Agent 无法 KYC |

**关键数据**：
- Solana x402 交易量 > Base，因为 gas 低 20-50 倍
- API SaaS 市场明年 6000 亿美元，Jay Yu（Pantera）base case 25% Agent 驱动 = 1500 亿美元
- 目前真实 volume 还很小，但 Jay 认为"很正常"——ChatGPT 月活也是指数级爆发

**投资关注点**：
- 协议是否有真实交易量（不是 wash trading）
- gas 成本是否足够低（micro payment 场景）
- 是否支持多链（不要押注单链）

### 方向 2：Agent 身份认证（Identity）

**核心逻辑**：
- 没有身份就没有信贷，没有信贷就没有真正的 Agent 经济
- KYC 对 Agent 不适用，需要新的身份范式

**细分方向**：

| 方向 | 描述 | 技术方案 | 代表项目 |
|------|------|----------|----------|
| **ZK 身份** | 证明身份属性而不透露真实信息 | ZK Proof | ZKID（Jay Yu 黑客松项目） |
| **社交身份** | 基于社交媒体账户的身份绑定 | OAuth + 链上凭证 | 基于 TikTok/Instagram 绑定 |
| **Agent Passport** | 用户ID + Agent ID + 会话ID 三层架构 | 智能合约 | Kite AI（EP15 嘉宾项目） |
| **KYA (Know Your Agent)** | 从 KYC 到 KYA，了解你的 Agent | 链上行为数据 + 评分 | 早期概念 |

**关键洞察**：
- Jay Yu：ZK 是加密身份认证的「USB-C」——标准化、可插拔、跨平台
- 发展中国家的 Creator 没有银行账户，但可以通过 TikTok/Instagram 绑定获得类似 KYC 的钱包
- 这不是替代 KYC，而是**扩展 KYC 的边界**

**投资关注点**：
- 身份体系是否被实际采用（有真实用户/Agent）
- 是否兼容现有身份基础设施（KYC、OAuth 等）
- 隐私保护能力（ZK 的工程化成熟度）

### 方向 3：Agent 安全护栏（Security Guardrail）

**核心逻辑**：
- Prompt Injection 可以把 Agent 的 API Key 全部偷走发到暗网
- 安全不是可选项，是 Agent 经济的生死线

**细分方向**：

| 方向 | 描述 | 当前状态 | 机会大小 |
|------|------|----------|----------|
| **Prompt Injection 防护** | 防止恶意输入劫持 Agent | 已有 prototype | ⭐⭐⭐⭐ |
| **API Key 隔离** | 搜索功能与支付功能隔离 | 设计空间探索中 | ⭐⭐⭐⭐ |
| **MPC 密钥管理** | 多方计算保护私钥/API Key | Privy 等已有方案 | ⭐⭐⭐ |
| **TEE 可信执行** | 硬件级安全隔离 | Jay Yu 认为 80% 用户只需 TEE | ⭐⭐⭐ |
| **Agent 行为审计** | 记录 Agent 所有行为，可追溯 | 早期 | ⭐⭐⭐⭐ |
| **沙箱环境** | Agent 在隔离环境中执行 | 虚拟机/容器方案 | ⭐⭐⭐ |

**关键洞察**：
- Jay Yu：私钥本质上就是 API Key，钱包的安全体系完全可以迁移到 API Key 管理
- API Key 比私钥更好管理：可以 rotate、可以做 scoping、leak 了可以后端关掉
- 但 Prompt Injection 是目前最大的安全隐患，已有 prototype 证明可行

**投资关注点**：
- 是否有真实攻击案例和防护方案
- 是否兼容主流 Agent 框架（OpenClaw、LangChain 等）
- 性能开销（安全不能牺牲用户体验）

### 方向 4：Agent 信贷 / 借贷市场

**核心逻辑**：
- 传统银行最赚钱的是信贷，Agent 银行也不会例外
- 现在有开发者发现：Agent 访问服务但付不了钱——钱包里没钱

**市场机会**：
- Jay Yu：Agent 贷款市场"会是一个非常大的机会"
- 未来 12-18 个月会有更多尝试
- 前提是：完善的 Agent 身份体系

**细分方向**：

| 方向 | 描述 | 类比传统金融 |
|------|------|-------------|
| **副卡模式** | 基于用户信用给 Agent 开副卡 | 信用卡副卡 |
| **Agent 信用评分** | 基于 Agent 行为数据评估信用 | 花呗/借呗信用分 |
| **DeFi 借贷** | Agent 在链上 pool 借贷 | Aave/Compound |
| **供应链金融** | Agent 之间的信用链条 | 保理、应收账款 |

**关键洞察**：
- 给 Agent 发卡 = 给小朋友发副卡，看的是"父母"（用户）的还款能力
- 人可以靠大数法则评估信用，Agent 可能"突变"——需要看 Agent 周边架构
- 保险/违约概率计算是全新课题，数据点太少

**投资关注点**：
- 身份体系是否完善（没有身份就没有信贷）
- 是否有真实借贷需求（不是伪需求）
- 违约回收机制（Agent 的"催收"怎么玩？）

### 方向 5：稳定币银行 / 新型银行（Neobank）

**核心逻辑**：
- 稳定币是全球性的「支付宝」——7x24、跨境、即时结算
- 但稳定币本身不产生利息，需要"银行"来提供增值服务

**市场机会**：
- Circle 正在申请银行牌照
- AllScale（EP17 嘉宾项目）：自托管稳定币 Neobank，从发票切入，1200万美元收款
- 发展中国家的 Airbnb host 更愿意收稳定币而非当地货币（payout 周期太长）

**细分方向**：

| 方向 | 描述 | 代表项目 |
|------|------|----------|
| **稳定币收付款** | 企业用稳定币收发全球款项 | AllScale、Interlace |
| **稳定币生息** | 为稳定币余额提供收益 | 各大 DeFi 协议 |
| **稳定币发卡** | 稳定币余额绑定实体卡消费 | 多家公司在做 |
| **稳定币 payroll** | 用稳定币发工资 | 远程工作趋势驱动 |
| **稳定币 treasury** | 企业财库管理 | 跨境企业需求 |

**投资关注点**：
- 合规能力（牌照、KYC/AML）
- 银行关系（能否对接传统银行做出入金）
- 用户真实需求（不是 Crypto 内循环）

### 方向 6：Agent 发现层 / Agent 经济基础设施

**核心逻辑**：
- Agent 不知道哪个服务最好，需要"发现"机制
- 不是搜索引擎，而是"菜谱"式的 workflow 编排

**细分方向**：

| 方向 | 描述 | 代表项目 |
|------|------|----------|
| **Agent SEO** | 如何在 Agent 搜索中被推荐 | 早期概念 |
| **Agent 广告** | 针对 Agent 的广告投放 | 全新市场 |
| **Workflow 编排** | 把多个 API call 绑定成"菜谱" | 8004Scan（YQ 项目） |
| **Agent 评分/声誉** | 链上不可篡改的 Agent 评分 | 8004Scan |
| **Agent APP Store** | Agent 技能市场 | 多家公司在做 |

**关键洞察**：
- Jay Yu：发现层需要"菜谱"形式——三四个 API call 绑定成一个 workflow，说明 cost、操作、规则
- 传统搜索引擎给的是列表，Agent 需要的是**最优 workflow**
- Agent 广告是全新市场：为什么 Agent 推荐 A 而非 B？

**投资关注点**：
- 是否有真实 Agent 在使用
- 评分/声誉系统是否抗刷榜
- 商业模式（按调用收费？按排名收费？）

### 方向 7：RWA（真实资产代币化）

**核心逻辑**：
- 不是所有资产都适合上链，但适合上链的资产体量巨大
- 不是"为了上链而上链"，而是"链上能带来什么传统金融做不到的价值"

**市场数据**：
- DTC（DTCC 子公司）托管资产超 100 万亿美元
- SEC 给 DTC "不采取行动函"，允许代币化试点
- 香港、新加坡、欧盟都在推进 RWA 监管框架

**细分方向**：

| 方向 | 描述 | 成熟度 |
|------|------|--------|
| **代币化基金** | 基金份额上链 | 已有案例（Tokeny、Securitize） |
| **代币化房地产** | 房地产碎片化投资 | 早期 |
| **代币化债券** | 政府/企业债券上链 | 试点中 |
| **代币化股票** | 股票 7x24 交易 | DTC 试点中 |
| **供应链金融** | 应收账款代币化 | 较成熟 |

**投资关注点**：
- 监管合规（各国政策差异大）
- 流动性（上了链不等于有流动性）
- 真实需求（是谁需要这个？）

## 三、国内独特机会

### 3.1 香港 Web3 政策窗口

- 香港稳定币监管框架推进中
- 香港 Web3 嘉年华（2026 年 4 月）成为亚洲最大行业峰会
- 机会：合规稳定币发行、香港持牌交易所、RWA 平台

### 3.2 跨境支付 / 出海企业

- 中国企业出海（电商、游戏、SaaS）需要低成本跨境支付
- 稳定币结算 vs 传统 SWIFT：分钟级 vs 天级
- 机会：稳定币收付款基础设施、跨境 payroll

### 3.3 发展中国家金融普惠

- 全球 20 亿人无银行账户
- 稳定币 + AI Agent 可以让这些人直接接入全球经济
- 机会：非 KYC 身份绑定、Creator Payout、微支付
