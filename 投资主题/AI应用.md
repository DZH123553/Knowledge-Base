## 一、从第一性原理出发

### 1.1 AI 应用的本质是什么？

大模型是电，AI 应用是电器。电力本身不产生价值，电灯、冰箱、洗衣机才产生价值。

同理：
- **GPT-4/o1/Claude 是电**（基础设施，越来越 commodity）
- **AI 应用是电器**（差异化在于场景理解、workflow 嵌入、用户数据飞轮）

**第一性结论**：AI 应用的投资价值 = （场景深度 × 数据壁垒）/ 模型依赖度

模型依赖度越低、场景越深、数据壁垒越高 → 投资价值越高

### 1.2 为什么现在是 AI 应用层的黄金窗口？

| 阶段 | 时间 | 特征 | 机会 |
|------|------|------|------|
| **模型层** | 2023-2024 | 参数战争、算力竞赛 | 已被 OpenAI/Anthropic/智谱等占领 |
| ** infra 层** | 2024-2025 | 向量数据库、RAG、Agent 框架 | 同质化严重，资本效率下降 |
| **应用层** | 2026-? | 垂直落地、ROI 验证、商业模式闭环 | **当前窗口** |

> a16z、Menlo、Bessemer 等顶级 VC 2026 年预测一致：**2026 是 AI 的 ROI 回报年，CFO 将取代用户成为真正的买家。**

## 二、细分方向发散

### 方向 1：Agent 作为"员工"而非"工具"

**核心逻辑**：
- 从"帮我查个资料"（工具）→ "帮我处理完这个退款"（员工）
- Agent 需要有预算、权限、审计日志、升级规则
- 定价从 "$/seat" 转向 "$/outcome"

**参考标的/模式**：
- **客服 Agent**：自动处理退款、改签、投诉（已有 volume，但质量参差不齐）
- **销售 Agent**：自动跟进线索、生成提案、预约会议
- **HR Agent**：简历筛选、面试安排、入职流程
- **财务 Agent**：发票处理、对账、费用报销

**知名 VC 动向**：
- a16z 2026 年重点：agents move from tools to employees
- Menlo：tiny teams reach outsized revenue as AI decouples growth from headcount
- 红杉中国 2026 Q1：Floatboat（AI Agent 原生工作环境平台，200万美元种子轮）

**投资关注点**：
- 是否有明确的 outcome metric（ticket resolved、$ recovered、invoice processed）
- 是否有权限控制和审计能力（CFO 会关心的）
- 是否有人类 in the loop 的 fallback 机制

---

### 方向 2：垂直行业 AI（Vertical AI）

**核心逻辑**：
- 通用 AI 解决通用问题，垂直 AI 解决行业 know-how 问题
- 行业数据 + 行业 workflow = 壁垒

**细分赛道**：

| 行业 | 痛点 | AI 应用形态 | 国内机会 |
|------|------|------------|----------|
| **法律** | 合同审查慢、贵 | 合同自动审查 + 风险点标注 | 中（法务 SaaS 已有基础） |
| **医疗** | 诊断资源不均、病历处理量大 | 病历结构化 + 辅助诊断 + 患者随访 | 高（政策敏感，需合规） |
| **金融** | 研报生成、风控、合规 | 研报自动生成 + 舆情监控 + KYC | 高（但监管严格） |
| **教育** | 个性化教学成本高 | AI 家教 + 自适应学习路径 | 中（双减后需避开学科培训） |
| **建筑/工程** | 图纸审查、造价估算 | 图纸自动审图 + 工程量估算 | 高（BIM 数据基础好） |
| **零售/电商** | 选品、定价、客服 | 智能选品 + 动态定价 + 智能客服 | 高（数据丰富） |

**知名 VC 动向**：
- a16z & Sequoia 2026：healthcare、finance、generative AI applications
- 高瓴/红杉中国 2026 Q1：AI 与实体产业深度融合，垂直行业应用

### 方向 3：AI 原生硬件（AI Native Hardware）

**核心逻辑**：
- 软件定义硬件 → AI 定义硬件
- 不是"给硬件加 AI"，而是"为 AI 重新设计硬件"

**细分方向**：

| 形态 | 场景 | 代表案例 |
|------|------|----------|
| **AI 穿戴设备** | 健康监测、运动追踪 | OdyssLife 智能项链（红杉中国领投，2亿元） |
| **AI 办公设备** | 人机协同工作空间 | Floatboat（红杉中国种子轮） |
| **AI 教育硬件** | 语言学习、编程启蒙 | 已有多家公司 |
| **AI 内容创作设备** | 相机、录音笔 | 大览科技 AI 长焦相机（红杉中国领投） |
| **AI 陪伴设备** | 情感陪伴、儿童教育 | 处于早期 |

**投资关注点**：
- 硬件是否"AI Native"（为 AI 重新设计），而非"AI + 传统硬件"
- 是否有清晰的软件服务收入（hardware as a trojan horse for SaaS）
- 供应链能力（中国优势）

### 方向 4：Agent Commerce / AI 电商

**核心逻辑**：
- 从"人找货" → "Agent 帮人找货" → "Agent 之间交易"
- 广告形式迭代：从"给人的广告" → "给 Agent 的广告"

**细分方向**：

| 层级 | 描述 | 机会大小 |
|------|------|----------|
| **Micro payment / API** | Agent 调用 API 付费（x402） | ⭐⭐⭐⭐⭐ 最确定 |
| **Macro payment / 电商** | Agent 帮用户买 Amazon/DoorDash | ⭐⭐⭐ 早期 |
| **Agent-to-Agent 交易** | Agent 之间自主交易 | ⭐⭐ 非常早期 |
| **Agent SEO/广告** | 如何在 Agent 搜索中被推荐 | ⭐⭐⭐⭐ 全新市场 |

**参考数据**：
- API SaaS 市场明年约 6000 亿美元，Pantera 的 Jay Yu 预测 25% 由 Agent 驱动 = 1500 亿美元
- 知名 VC 动向：a16z 关注 AI-driven automation tools、decentralized AI infrastructure

---

### 方向 5：情感 AI / 陪伴 AI

**核心逻辑**：
- AI 不仅是效率工具，也是情感载体
- 从"解决问题" → "陪伴存在"

**细分方向**：
- **AI 伴侣/朋友**：Character.AI、Replika 已验证需求，但商业化还在探索
- **AI 心理健康**：心理咨询 AI、冥想引导 AI
- **Empathetic AI**：Humans&（a16z 投资，4.8亿美元早期轮）——通过长时程强化学习放大人类协作和情感连接

**投资关注点**：
- 安全边界（safety、boundaries、emotional UX）比智能更重要
- 监管风险（未成年人保护、成瘾性）
- 变现模式（订阅制？情绪价值如何定价？）

---

### 方向 6：AI 基础设施的"最后一公里"

**核心逻辑**：
- 大模型和 Agent 框架已经成熟，但"用起来"还有很多 friction
- 谁解决"最后一公里"，谁就能捕获价值

**细分方向**：

| 痛点 | 解决方案 | 代表公司/模式 |
|------|----------|--------------|
| 手动注册 API、放 .env | Agent 自动发现 + 调用 API | API orchestration 平台 |
| Agent 技能管理混乱 | Skill file management / Agent APP Store | 8004Scan（YQ 项目） |
| Agent 不知道用哪个工具 | Wallet / Tool Orchestration | Moonpay 标准提案 |
| Agent 搜索发现效率低 | Agent SEO / "菜谱"式 workflow | 早期 |
| Agent 评估困难 | Agent evaluation / monitoring / fail-safes | 基础设施机会 |

## 三、国内独特机会

### 3.1 "AI + 制造业"（中国优势）

- 中国制造业体量全球最大，AI + 质检、AI + 排产、AI + 供应链优化
- 不是"替代工人"，而是"让工人更值钱"
- 标杆：宁德时代已经引入人形机器人产线

### 3.2 "AI + 出海"

- 中国企业出海（电商、游戏、SaaS）需要本地化 AI 工具
- 语言、文化、合规的 AI 化
- 机会：AI 本地化工具、AI 跨境客服、AI 海外营销

### 3.3 "AI + 银发经济"

- 中国老龄化速度全球最快
- AI 健康监测、AI 陪伴、AI 远程医疗
- 支付能力强，但产品形态需重新设计

## 四、反方向（不投什么）

| 类型 | 原因 |
|------|------|
| 纯模型层公司 | 已被巨头垄断，资本效率极低 |

| 纯概念无数据的行业 AI | 行业 know-how 需要时间和数据积累 |
