# UZI-Skill · 51人投资评审团实现原理深度拆解

> 基于 [wbh604/UZI-Skill](https://github.com/wbh604/UZI-Skill) v3.2.0 源码分析

---

## 一、整体架构：不是代码，是 Prompt Engineering

UZI-Skill 的 51 人评审团本质上是一个 **Prompt-driven Multi-Agent System**。它没有传统意义上的"打分函数"或"决策树代码"，而是通过三层结构实现的：

```
┌─────────────────────────────────────────────────────────────┐
│  Layer 1: YAML Persona 档案（51个）                          │
│  → 每个投资者的投资哲学、关键指标、回避事项、语言风格           │
├─────────────────────────────────────────────────────────────┤
│  Layer 2: Rules 引擎骨架分                                    │
│  → 22维数据输入 → 按各组规则计算初步 score/signal             │
├─────────────────────────────────────────────────────────────┤
│  Layer 3: LLM Role-play 覆盖                                 │
│  → Spawn 4个并行 sub-agent 分组扮演 → 覆盖 headline/reasoning │
└─────────────────────────────────────────────────────────────┘
```

**关键洞察**：脚本（Rules 引擎）负责"算数"，Agent（LLM role-play）负责"下判断"。这是这个系统最精妙的地方——它不靠硬编码规则出最终分，而是靠 LLM 真正"扮演"投资者去推理。

## 二、51人的组织方式：7大流派

### 分组逻辑

| 组 | 流派 | 人数 | 核心方法论 | 代表人物 |
|----|------|------|-----------|---------|
| A | 经典价值 | 6 | ROE/护城河/安全边际/FCF | 巴菲特、格雷厄姆、费雪、芒格、邓普顿、卡拉曼 |
| B | 成长投资 | 4 | PEG/营收增速/颠覆潜力 | 林奇、欧奈尔、蒂尔、木头姐 |
| C | 宏观对冲 | 5 | 宏观周期/反身性/非对称押注 | 索罗斯、达里奥、马克斯、德鲁肯米勒、罗伯逊 |
| D | 技术趋势 | 4 | Stage 2确认/量能/均线/箱体突破 | 利弗莫尔、米内尔维尼、达瓦斯、江恩 |
| E | 中国价投 | 6 | 好生意+好价格/长期持有/行业拐点 | 段永平、张坤、朱少醒、谢治宇、冯柳、邓晓峰 |
| F | A股游资 | 23 | 龙虎榜/涨停板/板块龙头/T+1动量 | 章盟主、赵老哥、炒股养家、佛山无影脚… |
| G | 量化系统 | 3 | 多因子（动量/价值/质量/波动率） | 西蒙斯、索普、大卫·肖 |

**总计 51 人**（注意 README 写"50"是早期版本，实际 personas/ 目录有 51 个 YAML 文件）。

### 为什么这样分组？

1. **方法论同质化组内聚合**：同一组的投资者评判标准相近，可以共享 reference 文件
2. **方法论异质化组间冲突**：价值派和游资对同一只票的评分天然会两极分化，这种冲突本身就是信息
3. **A股特殊性**：F组23人专门处理A股游资视角，非A股直接 skip——避免方法论错配

## 三、Persona YAML 档案格式

每个投资者一个 YAML 文件，这是整个系统的"单源真相"（single source of truth）。

### 以巴菲特为例（buffett.yaml）

```yaml
id: buffett
name: 沃伦·巴菲特
school: value_classic
group: A
nationality: 美国
era: 1956-今

philosophy: |
  以合理价格买伟大公司，不是以伟大价格买合理公司。
  看得懂的生意、护城河、诚实管理层、可理解的经济学。
  能力圈外的一律不碰——科技股、商品股、复杂衍生品全在外面。
  安全边际永远在，宁可错过不能错买。

key_metrics:
  - ROE 连续 10 年 > 15%（越稳定越好）
  - 自由现金流 / 所有者盈余 充裕且持续增长
  - 低负债（长债 / 权益 < 1）
  - 管理层持股 + 现金分红政策透明
  - 量化的护城河（品牌溢价 / 规模成本 / 切换成本）
  - PE < 25（贵茅台 2007 年 40 倍我也减仓了）

avoids:
  - 高估值成长股（无盈利的不碰）
  - 需要持续大额资本再投入的生意
  - 快速变化的科技行业（IBM 那次是我少有的失误）
  - 杠杆 / 衍生品 / 复杂结构
  - 能力圈外的任何标的

a_share_view: |
  A股里只有极少数真正符合我标准的——茅台、泸州老窖、格力早期、福耀。
  国资实控股大多不透明、不尊重小股东、频繁资本运作，我基本都回避。

voice: |
  朴素、平民化、中西部口音风格。喜欢用农场 / 棒球 / 吃汉堡的比喻。
  引格雷厄姆和芒格原话。拒绝预测短期价格。以十年为单位思考。
  常说 "owner earnings" / "margin of safety" / "circle of competence"。

famous_positions:
  - 可口可乐（1988 买入持有至今）· 护城河+品牌+全球消费
  - 苹果（2016 买入至今）· 芒格说服的例外
  - 比亚迪（2008 借芒格推荐买入，2022 开始减持）
```

```yaml
id: zhao_lg
name: 赵老哥（赵强）
school: youzi_faction
group: F

key_metrics:
  - 封板时间（早封 vs 尾盘封）· 早封 10:00 前 = A 级

voice: |
  短句、北方话、偶尔北京骂街风。
  话少信息密度高。不解释哲学 · 只说操作："买"、"跑"、"不碰"。
```

### Persona 设计的 6 个维度

| 维度 | 作用 |
|------|------|
| `philosophy` | 投资哲学的自然语言描述，决定"他会怎么想" |
| `key_metrics` | 他看哪些指标、阈值是多少，决定"他会怎么算" |
| `avoids` | 他坚决不碰什么，决定"什么情况下给 skip 或 0 分" |
| `voice` | 语言风格标签，决定 comment 字段的口吻 |
| `famous_positions` | 历史真实持仓，用于"real holdings trump rules"覆盖 |
| `a_share_view` | A股特殊视角，决定他对A股市场的整体态度 |

## 四、Rules 引擎骨架分

### 输入

- `.cache/{ticker}/dimensions.json` — 19维评分（从22维原始数据计算而来）
- `.cache/{ticker}/raw_data.json` — 22维原始数据

### 处理逻辑

对每位投资者：
1. 取出 `fields` 白名单（该投资者关心哪些维度）
2. 从 dimensions.json 提取相关字段
3. 按该投资者的 `key_metrics` 规则逐一检查
4. 计算初步 score / signal / verdict

### 输出格式（Pydantic Signal）

```json
{
  "investor_id": "buffett",
  "name": "巴菲特",
  "group": "A",
  "avatar": "avatars/buffett.svg",
  "signal": "bullish | neutral | bearish",
  "confidence": 87,
  "score": 82,
  "verdict": "强烈买入 | 买入 | 关注 | 观望 | 等待 | 回避 | 不达标 | 不适合",
  "reasoning": "1-3句具体逻辑",
  "comment": "用该投资者语言风格的金句1-2句",
  "pass": ["..."],
  "fail": ["..."],
  "ideal_price": 16.20,
  "period": "3-5年"
}
```

### Confidence 校准规则

| 区间 | 含义 |
|------|------|
| 85-100 | 核心方法论硬指标全部命中或全部不命中 |
| 60-84 | 多数命中 |
| 30-59 | 部分命中、需要等待信号 |
| 0-29 | 方法论不适用此股 / 信息不足 |

## 五、LLM Role-play 覆盖（最关键的一步）

Rules 引擎的骨架分只是"参考"。真正的评分来自 LLM 的 role-play。

### 执行流程

```
Stage 1（脚本）→ 骨架分
   ↓
Agent 介入（必须）
   1. 读 panel.json 中 51 人的骨架分
   2. Spawn 4 个并行 sub-agent，每组 role-play
   3. 用 agent 判断覆盖 panel.json 的 headline/reasoning/score
   4. 写 agent_analysis.json
   ↓
Stage 2（脚本）→ 合并生成报告
```

### Role-play 规则（agents/investor-panel.md）

1. **Think AS the investor, not ABOUT the investor**
   - ❌ "Buffett would think X"
   - ✅ "ROE 6.6% is well below my 15% threshold"

2. **You CAN override the rule engine score**
   - 如果巴菲特实际持有这只股票 → override 到 85+
   - 如果格雷厄姆规则给50但PE是33 → override 到 15

3. **Real holdings trump rules**
   - 已知持仓是最强信号

4. **Industry affinity matters**
   - 木头姐分析传统建筑 = skip 或极低分
   - 格雷厄姆分析生物科技股 = bearish 无论PE

5. **Headlines MUST cite numbers**
   - ❌ "looks good"
   - ✅ "ROE 18.2% exceeds my 15% bar for 5 consecutive years"

6. **Skip is valid**
   - 游资评美股、木头姐评白酒 = signal="skip"

### 12 个 Flagship Persona（v2.15 新增）

从 v2.15 起，12 个核心投资者有手写 YAML，其余 39 个是 auto_generated_stub：

**Flagship（手写）**：巴菲特、芒格、格雷厄姆、费雪、林奇、木头姐、索罗斯、达里奥、段永平、张坤、赵老哥、章盟主

**Stub（自动生成）**：其余 39 位，Rules 引擎输出优先，YAML 仅补充身份信息

**硬要求**：

- 每条 reasoning 必须带 `voice` 字段的风格词
- **signal 必须与 persona 历史立场对齐**：巴菲特不会对 PE 882 的股票说买入；赵老哥不会对 9000 亿市值说"打板"

## 六、游资射程预过滤（F组特殊机制）

这是 UZI 最独特的设计之一。

### 问题

23位游资风格差异极大：
- 有人专做首板（N=1）
- 有人专做连板（N=2-3）
- 有人只玩小市值（<100亿）
- 有人只做特定题材（芯片/新能源/消费）

如果让所有人对所有A股打分，会产生大量噪音。

### 解决方案：`lib/seat_db.py` — 射程规则

```python
from lib.seat_db import SEATS, is_in_range

# 对22位游资，先用 is_in_range(nickname, ticker_features) 判断是否在射程内
# - 在射程 → 正常评分
# - 不在射程 → signal: "neutral", verdict: "不适合", confidence: 90
```

**射程维度**：
- 市值区间（小盘/中盘/大盘）
- 板数偏好（首板/连板/高位）
- 题材偏好（芯片/新能源/消费/金融...）
- 地域偏好（江浙/广东/北京...）

**效果**：22位游资中通常有 N 位返回"不适合"，只有少数真正在射程内的才会认真评分。

## 七、核心评分公式

### 公式（v2.11 校准版）

```
overall = fund_score × 0.6 + consensus × 0.4

consensus = (bullish_count + 0.6 × neutral_count) / active_count
```

### 参数说明

| 参数 | 来源 | 说明 |
|------|------|------|
| `fund_score` | 22维基本面评分 | 财务/估值/行业等量化维度 |
| `consensus` | 51评委投票统计 | 多空倾向的聚合 |
| `0.6` | neutral 权重 | v2.11 从 0.5 上调到 0.6 |

### 为什么 neutral 权重是 0.6？

**v2.11 校准记录**：

> 51评委里价值派+游资 35人偏保守，neutral 权重 0.5 让白马 consensus 仅 37，0.6 更贴近"不坑但不是心头好"的真实语义。

**典型白马（茅台）**：
- v2.9.1：`fund=62, consensus=45 → overall=55 → 观望优先`
- v2.11：`fund=62, consensus=50 → overall=57 → 观望优先`（更接近"可以蹲一蹲"边界）

### Verdict 阈值（v2.11 校准）

| 阈值 | 档位 | 含义 |
|------|------|------|
| ≥80 | 强烈买入 | 以前从未有股能≥85，所以下调5分让白马能进档 |
| ≥65 | 买入 | |
| ≥50 | 关注 | |
| ≥35 | 观望 | |
| <35 | 回避 | 真正的坑仍会 <35，辨识度不降反升 |

## 八、风格动态加权（v2.7）

不是固定权重，而是按股票类型动态调整：

### 股票风格检测

系统自动识别股票属于哪种风格：
- 白马 / 高成长 / 周期 / 小盘投机 / 分红防御 / 困境反转 / 量化因子 / 中性兜底

### 动态调整内容

1. **51评委组级权重**：A-G × style 矩阵 + 8个个体 override
2. **22维 fundamental dim multiplier**：不同风格看重不同维度
3. **neutral 半权计入 consensus**：修正旧公式 0% 权重的问题

### 量化因子型检测

```python
# lib/quant_signal.detect_quant_signal
# 结构性特征：基金 top-1 持仓 < 2% → 疑似量化（无需名字含"量化"）
# 持有目标股票的基金里 ≥3家量化基金且把目标股放进 top-10 → quant_factor
```

## 九、The Great Divide · 多空大分歧

当51评委出现极端分歧时（如费雪100分 vs 卡拉曼96分），系统会生成"多空大辩论"：

```json
{
  "great_divide_override": {
    "punchline": "DCF说高估23%，但城投重组预期让LBO视角的IRR仍有18%",
    "bull_say_rounds": ["理由1", "理由2", "理由3"],
    "bear_say_rounds": ["理由1", "理由2", "理由3"]
  }
}
```

**设计意图**：分歧本身是信息。报告不"和稀泥"，而是把冲突写进报告。

## 十、自查 Gate（v2.9 机械强制）

`assemble_report.py::assemble()` 会自动跑 `lib/self_review.py` 检查 ~13 条规则：

| severity | check | 背后 BUG |
|----------|-------|---------|
| 🔴 critical | `check_panel_non_empty` | panel全skip/avg_score异常 |
| 🔴 critical | `check_all_dims_exist` | wave2 timeout导致维度缺失 |

| 🟡 warning | `check_industry_data_coverage` | 定性字段需web_search补 |

## 十一、可借鉴到自己的系统

### 1. Persona 档案化（最重要）

把投资人的方法论写成结构化 YAML：
- `philosophy` → 决定"他会怎么想"
- `key_metrics` → 决定"他会怎么算"
- `avoids` → 决定"什么情况下 skip"
- `voice` → 决定输出的语言风格

比纯 prompt 更稳定、更可维护、可审计。

### 2. 骨架分 + Agent覆盖的双层架构

- Layer 1: 规则引擎快速出骨架分（确定性、可重复）
- Layer 2: LLM role-play 覆盖真正判断（灵活性、创造性）
- 两层互补，不是替代

### 3. 射程预过滤

不是所有 Agent 都要评所有标的。按风格/能力圈/市场范围预过滤，大幅降低噪音。

### 4. Consensus 中性权重

neutral 不是0分，而是0.6权重。避免保守派拉低 consensus，同时保持对极端信号的敏感度。

### 5. 风格动态加权

按标的类型（白马/成长/周期/量化）动态调整各组权重，而不是一刀切。

### 6. 分歧即信息

不要把多空分歧"和稀泥"平均掉。把分歧本身写进报告，让用户看到"谁在看多、谁在看空、为什么"。

## 十二、局限与注意事项

1. **Persona 可能 hallucinate**：39个 stub persona 主要靠 Rules 引擎，LLM 可能编造不存在的历史言论
2. **A股数据依赖**：akshare/xueqiu/东财等数据源不稳定，需要多层 fallback
3. **Token 消耗大**：51人 role-play 在中等深度下已很费 token，deep 模式更贵
4. **非代码驱动**：整个系统高度依赖 LLM 的 role-play 质量，不同模型效果差异大
5. **v2.15 才修硬伤**：早期版本有"林奇对 EPS 0 的股票说 PEG 可算"这类明显 bug
